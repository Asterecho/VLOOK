概述

本产品定位以直播的方式进行非标商品销售的服务平台。

本产品主要提供直播服务、商品展示、信息沟通、支付、订单管理、秩序管理等服务，而具体的直播内容、销售的商品、商品的发货、售后等则由入驻的主播提供。

名词解释

名词说明
非标商品包括但不限于：手工艺、烹饪、木艺、雕刻、文玩、字画、收藏品、鉴定服务、培训等
闪购针对直播过程销售非标商品的销售与购买服务，采用由买方出价，卖方确认的商品销售方式
基于这种销售方式可以衍生出多种场景，包括但不限于：购买固定价格商品/服务、非公开的竞价拍卖
竞拍以公开竞价的方式进行商品拍卖，由主播管理拍卖过程，观众参与出价，出价最高者获得竞拍商品的购买权
虚拟资金账户指主播在平台上用于记账的虚拟资金账户，包括可提现账户、担保支付账户、押金账户等
可提现账户只有该账户中的资金才可被提现
担保支付账户该账户中的资金不能提现，确认收货后资金会转入到可提现账户
押金账户用于竞拍等场要求的押金
直播间客服指入驻主播的助理，只针对主播自己业务提供客服咨询
平台客服本应用的官方客服
字/字符一般情况下指中文字符（英文、数字均视为中文字符）
主要涉及文档中有关文本长度数量限制等场合

基础

帐号服务

用户帐号体系延用YY现有的用户帐号体系。

注册与登录

业务规则：

  1. 用户可以在本平台上通过手机号注册新的YY帐号；
  2. 用户可通过已注册的YY帐号（YY号/YY通行证/手机号）登录，也可以通过第三方帐号登录，目前支持微信方式登录；
  3. 成功登录后自动保存帐号信息（YY号/YY通行证/手机号、密码），下次登录时能在登录页面直接选择并快速登录，支持手工开删除已保存的帐号信息；
  4. 同一个帐号不能在两个或以上设备上同时登录本应用，当同一个帐号在其它设备上登录后，原登录设置需要及时强制下线，并通知用户。

找回密码

支持用户通过绑定YY帐号中绑定的手机号找回密码。

个人信息

业务规则：

  1. 个人信息包括昵称、签名、头像、YY号；
  2. 修改的个人信息同步到对应的YY帐号中。

个人信息如下：

信息项必填默认值来源说明
头像Y默认头像用户上传可从相册、拍照形式获取图像后上传
昵称Y-用户输入最多输入20个字
签名N-用户输入最多输入20个字
YY号Y-系统生成只读。显示当前帐号对应的YY号

绑定手机

如果用户是通过第三方登录，或原有YY帐号没有绑定手机号的情况下，可在登录后通过该功能绑定手机号。绑定手机号后才能进行申请实名认证。

实名认证

实名认证使用YY UDB的实名认证方案，注册用户需要完成实名认证才能开始直播。

消息中心

业务规则：

  1. 消息中心入口：直播首页、我的；
  2. 消息中心是汇聚应用内各类消息的主要入口；
  3. 消息中心内有未读的新消息时，则在消息中心入口显示相应的角标提示。

来自观众的咨询

业务规则：

  1. 只有在当前登录帐号是有效的直播间客服时，才显示该功能入口；
  2. 进入后显示所有向该帐号咨询的观众列表，消息列表以观众为单位进行组织；
  3. 列表排序方式以其发送/接收的最后一条消息的时间倒序排序，越晚发送/接收消息的客服排序越前；
  4. 观众有新咨询消息并且为未读时，则在对应的观众头像处显示角标提示；
  5. 点击进入列表项进入与对应观众的咨询会话页面，并可以随时继续咨询。

来自客服的消息

业务规则：

  1. 消息列表以直播间客服为单位进行组织；
  2. 列表排序方式以其发送/接收的最后一条消息的时间倒序排序，越晚发送/接收消息的客服排序越前；
  3. 直播间客服回复新消息并且为未读时，则在对应的直播间客服头像处显示角标提示；
  4. 点击列表项进入与对应客服的咨询会话页面，并可以随时继续咨询。

PUSH通知

PUSH通知分为应用内通知和系统级通知，具体如下：

通知类型说明
应用内通知本应用为当前程序时，接收到PUSH通知后，仅在应用内展示通知消息内容。点击消息执行业务具体指定的动作
展示规则：
- 通知展示时间为3秒，展示时间结束后不再显示；
- 如果同时有多条通知消息要展示，则按接收到通知的时间顺序展示。
系统级通知本应用退出后，当接收到PUSH通知时，仅以系统级通知形式展示。点击消息后先启动APP，然后再执行业务具体指定的动作

主播开播通知

业务规则：

  1. 当前帐号收到已关注主播的开播通知后，则展示该主播的开播通知；
  2. 展示的内容包括：主播头像、昵称、通知内容；
  3. 点击通知后执行：进入主播的直播间。

观众咨询通知

业务规则：

  1. 当前帐号收到观众发来的新消息，当前帐号所在页面不在消息列表或咨询会话中时，才触发应用内通知；
  2. 已退出应用，则展示系统级通知；
  3. 展示的内容包括：观众头像、昵称、消息内容；
  4. 点击通知后执行：进入与观众的咨询会话页面。

客服回复通知

业务规则：

  1. 当前帐号收到客服回复的新消息，当前帐号所在页面不在消息列表或咨询会话中时，才触发应用内通知；
  2. 展示的内容包括：观众头像、昵称、消息内容；
  3. 点击通知后执行：进入与观众的咨询会话页面。

启动屏

启动屏包括程序加载闪屏与宣传引导页，品牌信息与引导说明以图片形式提供，需要适配不同比例的分辨率，确保不变形，参考链接 →。

以上图片资源须直接打包到程序安装包中，不依赖网络能正常展示。

程序加载闪屏

只在程序完成加载前展示，多用于向用户展示产品品牌形象宣传图。

宣传引导页

业务规则：

  1. 程序完成加载后显示，用于展示产品的特性、功能等介绍信息，包括1个或多个页面，滑动切换。在最后一页中提供进入APP的入口。
  2. 同一程序版本的引导信息只在首次启动时显示一次，如果卸载程序或安装新的程序版本，则重置该条件。

强制升级

强制升级一般用于旧版本存在安全隐患、原有服务已下线、新功能推广等情况。

程序加载完成后检测是否需要强制升级，如需强制升级则弹出升级提示，用户在没有完成强制升级最新版本前，产品的所有功能都不可用。

不同平台的引导策略有如下区别：

平台引导策略
Android公司平台上发布了更新的版本，则引导用户完成升级程序包的下载、安装与重新启动
iOS应用商店有更新的版本，则引导用户跳转至AppStore，由用户通过AppStore完成程序的更新操作

直播购物

直播场景下的非标商品销售与传统的标品电商有比较大的差别，为了能让主播从多样繁杂的商品管理中解脱出来，让直播购物的体验与与线下面对面购物交易更为接近，所以设计了针对直播时非标商品的销售与购买方式闪购。

闪购具有以下特点：

  1. 由观众在直播过程中随时发起闪购申请，由主播处理是否接受，主播接受后再完成后续的支付和选择收件人操作；
  2. 主播不需要进行商品管理（商品图/详情页/库存/上下架），只需要适时开启闪购功能，并处理观众实时提交的闪购申请即可；
  3. 主播在处理闪购申请时还可根据实际情况对购物信息（单价、购买数量、是否出具证书、交易模式等）进行修改，以更好灵活地处理观众个性购物需求（如将直接支付改为担保支付等）；
  4. 在直播过程中可以查看待处理、未支付、已支付的订单。

支付

业务规则：

  1. 主播接受了观众的申请后，观众才能继续进行支付操作；

  2. 支持的第三方支付渠道：微信、支付宝；

  3. 仅支持以上支付渠道的余额、银行储蓄卡等形式进行支付；禁止以任何形式使用信用卡及信用卡衍生的支付方式进行支付；

  4. 支持直接支付和担保支付两种交易模式，主播可以通过在以下方式设置交易模式：

    • 直播过程中在闪购设置中设置闪购的交易模式，设置后对所有新提交的闪购订单都实时生效；
    • 主播在处理每笔闪购订单时，都可以再次设置当前订单的交易模式，该设置只对当前处理的订单有效。

  1. 直接支付：

    • 直播支持指观众支付的订单金额到账平台后，该笔金额直接记账为主播可提现部分；
    • 主播可随时提现该订单的资金。
  1. 担保支付：
  • 冻结观众支付的订单金额，待订单由观众确认签收后，再解冻对应资金；
  • 如果从发货之日起的第7个工作日都没有确认签收，则系统自动完成该笔订单的签收；
  • 冻结的资金不能进行提现，但须作为我的钱包余额的计算范围。

导入周期是60用户输入单位：分钟
  1. 导入规则：在「导入周期」内，为指定的「主播」当前开播的直播间内导入指定的虚拟观众数。其中，虚拟观众数在导入机制触发前由系统生成，具体数值为配置的最小与最大虚拟观众数之间的随机正整数；
  2. 虚拟观众导入后，在直播列表、直播间内能体现在线观众的数量变化。在观众列表、观众信息卡中能展示虚拟观众的昵称、头像、签名等信息；
  3. 在触发了导入虚拟观众机制后5分钟，再触发虚拟观众进出机制；
  4. 触发以上机制后，如果主播结束直播，则相应机制自动结束。

虚拟观众的YY通行证为：dianshang_XXXX，其中XXXX取值范围：0000～9999，共9998个（排除0001和5006）。

虚拟观众进出

虚拟观众进出是通过控制虚拟观众帐号进入或退出指定主播当前开播的直播间，模拟真实用户进出直播间的行为。

业务规则：

  1. 触发该机制后，在主播结束直播前需要持续模拟直播间内虚拟观众进出，规则为：每X秒进入m个虚拟观众，每Y秒并退出n个虚拟观众，其中：

    • X：80～120的随机正整数；
    • Y：80～120的随机正整数；
    • m：1～2的随机正整数；
    • n：1～2的随机正整数；
    • 以上X/Y/m/n的值，每5分钟重新生成一次。

  2. 虚拟观众信息和行为需要：

    • 在直播列表、直播间内体现数量变化；
    • 在观众列表中展示虚拟观众的昵称、头像；
    • 在直播间内展示进入、退出直播间的行为。
